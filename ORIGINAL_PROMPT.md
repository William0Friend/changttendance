9-LAYER FACE RECOGNITION PIPELINE
This pipeline is the core of the app. It must work in real classroom conditions including fluorescent overhead lighting, students wearing glasses and masks and hats, motion blur from students turning their heads, partial face occlusion, low-end laptop webcams, and rooms where some students are far from the camera. Implement every layer in recognition/layers.ts orchestrated by recognition/pipeline.ts.
Layer 1 is multi-scale detection. Run SSD MobileNet v1 at three input scales: the native frame size, 1.5x upscaled for students far from camera, and 0.75x downscaled for students close to camera. Merge detections across scales using non-maximum suppression to eliminate duplicate boxes. Reject any detection with score below 0.5. This catches faces that a single-scale pass misses entirely.
Layer 2 is pre-processing assessment. For each detected face crop the region from the video frame to a canvas. Compute average luminance by sampling pixels. Compute a sharpness estimate using Laplacian variance approximation by convolving a 3x3 Laplacian kernel over the grayscale crop and computing variance of the result. Record which conditions are poor: dark meaning luminance below 60, bright meaning luminance above 200, blurry meaning sharpness variance below threshold. Pass the crop and quality flags to Layer 3.
Layer 3 is adaptive pre-processing in recognition/preprocess.ts. Apply enhancements based on the quality flags from Layer 2. For dark frames apply histogram equalization by computing pixel value distribution and remapping to use full 0 to 255 range. For very dark frames apply CLAHE approximation by dividing the crop into a grid of tiles and equalizing each tile independently then interpolating boundaries. For blurry frames apply an unsharp mask by subtracting a Gaussian-blurred version of the crop from the original with a controlled gain. For fluorescent lighting apply warm correction by slightly boosting red channel and slightly reducing green channel to normalize the yellow-green cast common in university classrooms. Log which preprocessing steps were applied to the preprocessingApplied array for debugging and analytics.
Layer 4 is landmark detection. Run Face Landmark 68 Net on each pre-processed face crop. Compute landmark confidence as the detection quality score. Reject faces where landmark confidence is below 0.55. Extract the positions of the left eye center, right eye center, nose tip, left mouth corner, and right mouth corner as the five key alignment anchors.
Layer 5 is geometric alignment. Use the eye landmark positions to compute an affine transformation normalizing the face to a canonical upright frontal pose. Compute rotation angle from the vector between eye centers. Compute scale to normalize inter-eye distance to a standard 64 pixels. Compute translation to center the face. Apply the transformation to the pre-processed crop using canvas 2D transform. The output is a consistently sized and oriented face crop regardless of how the student is holding their head. This step is what separates demo-quality recognition from production-quality recognition.
Layer 6 is quality scoring in recognition/quality.ts. On the aligned crop compute a FaceQualityReport. Brightness score from average luminance after pre-processing. Sharpness score from Laplacian variance after pre-processing. Face size ratio from bounding box width divided by frame width, rejecting below 0.10 to handle students far from camera. Yaw estimate from horizontal asymmetry of facial landmarks, flagging above 30 degrees. Pitch estimate from vertical landmark positioning, flagging above 25 degrees. Overall quality score is a weighted sum: detection score 25 percent, landmark confidence 25 percent, sharpness 20 percent, brightness 15 percent, face size 10 percent, head angle penalty 5 percent. Assign FaceQualityGrade where Excellent is above 0.85, Good is above 0.70, Acceptable is above 0.55, and Rejected is below 0.55. During enrollment accept Acceptable or better. During attendance scanning use grade to weight the final confidence.
Layer 7 is descriptor extraction. Run Face Recognition Net ResNet-34 on the aligned quality-graded crop producing a 128-float FaceDescriptor. Only run this layer if quality grade is Acceptable or better to avoid wasting CPU on poor detections.
Layer 8 is matching and confidence calculation. Use FaceMatcher initialized once per session with all enrolled student embeddings. Never rebuild FaceMatcher during a scanning session as it is expensive. For each descriptor run findBestMatch to get euclidean distance and label. Compute a combined confidence score that is a weighted blend of FaceMatcher distance converted to similarity and quality grade score, where a high quality detection with distance 0.50 is more trustworthy than a low quality detection with distance 0.46. Apply the session recognition threshold to the combined confidence. Mark as low confidence if combined score is between threshold minus 0.1 and threshold. Mark as unknown if combined score is below threshold minus 0.1.
Layer 9 is temporal debouncing and ensemble voting in recognition/matcher.ts. Maintain a TemporalMatch map keyed by studentId tracking consecutive match count. Do not mark a student present on first match. Require 3 consecutive scan passes matching the same studentId before marking present. This eliminates false positives from motion blur and partial face detections completely. Track confirmed students in a Set for this session. Skip FaceMatcher for already-confirmed students on subsequent passes and just draw their cached bounding box, saving significant CPU for large classes. Count every frame where a face passes Layer 1 but fails to match any student after Layer 8 as an unknown face event and increment the session unknown face counter.
The full pipeline runs every 1000 milliseconds during attendance mode on a separate async chain from the canvas drawing loop. Use requestAnimationFrame at 60fps for overlay drawing only. Dispose all TensorFlow tensors after every detection pass using tf.dispose and tf.tidy to prevent memory leaks.
Adaptive threshold: track average combined confidence across all successful matches in the session. If average confidence has been consistently below 0.65 for 5 or more matches automatically relax the threshold by 0.05 and show a notification to the professor saying lighting conditions are poor and recognition sensitivity has been adjusted. Log adaptiveThresholdActive on the session record.
